1. Overview

Emura is a web-based manufacturing cost estimator built for estimators at contract manufacturing companies. It runs at emura.io and performs all calculations instantly in the browser — no waiting for server responses, no lag between input and result.

The application aggregates four cost categories across multiple finished goods and volume scenarios:

  • BOM / Material Costs — raw materials and purchased components, with volume-sensitive cost archive
  • Direct Labor — work-cell operations with cycle times, setup times, operator counts, and equipment costs
  • Indirect Labor — overhead categories (engineering, quality, purchasing, etc.) with annual, order-level, and line-level components
  • Subcontract Costs — outsourced operations or services, per-unit or fixed

Output is a cost-per-unit breakdown for every finished good at every volume scenario, with editable margin inputs and automatic sell price calculation.

1.1 Design Goals

  • Speed is a feature — all calculations run client-side; results update instantly on every input
  • Handles multi-FG quotes with different volume mixes per finished good
  • Clean separation of order-level vs. line-level setup costs throughout
  • Volume-intelligent material cost lookup with archive and proximity matching
  • All data auto-saves locally; export/import via JSON file for sharing or backup
  • Every input has a tooltip (ⓘ) explaining purpose and expected value
  • No pop-up warnings — errors shown inline; all structural changes are undoable

1.2 Non-Goals (current version)

  • ERP/MRP integration, CRM, inventory management, accounting
  • Multi-user collaboration, organizational accounts, and cloud sync are planned for future releases (see §8)

2. Key Concepts

2.1 Line vs. Order

LineAn individual production job for a single finished good. Each FG occupies its own line. Line setup time is specific to the FG and is incurred once per order event per FG.
OrderA build event where all lines run simultaneously — one line per FG. Order setup time is incurred once per event, and its cost is allocated across all units in the order (all FGs combined).

This distinction matters when sizing setup costs. A 30-minute order setup shared across 500 total units costs $0.05/unit at a $50/hr rate. A 30-minute line setup on an FG with only 100 units/build costs $0.25/unit — five times more — because it's not shared.

2.2 Volume Breaks

Volume break columns represent demand scenarios. Each column has a Builds Per Year (how many times per year all FGs are built) and an optional Target Total EAU. Each finished good independently has an EAU per column. The sum of FG EAUs is validated against the target and can be auto-populated from mix proportions.

2.3 EAU and Mix

EAU (Estimated Annual Usage) is the number of units of a given FG produced per year at a volume break. When a target total EAU is set on a break column, the system can auto-fill each FG's EAU by applying the same mix percentages from another column. A "Push mix" button copies proportions from any filled column to all other columns that have a target set.

2.4 Work Cell

A work cell is a direct labor operation with a fixed operator count and a single cycle time (seconds/unit). Cycle time governs throughput and must be ≤ takt time to meet annual demand on a single shift at the specified working hours.

2.5 Takt Time

Takt = (Working Hours/Year × 3,600) ÷ (Total Annual Units at the highest-demand break). Displayed as an informational banner on the Operations tab. Operations with cycle times exceeding takt are flagged by name in a warning banner. No operations are blocked — the flag is informational.

2.6 Equipment

Equipment is defined on the Equipment tab and assigned to one or more direct labor operations via a searchable selector. Each piece of equipment contributes up to three cost components per assigned operation:

  • CapEx: amortized capital cost, allocated by utilization % and depreciation years
  • Hourly Run Cost: variable cost per run-hour (cycle time only) — utilities, consumables, wear
  • Annual Maintenance: fixed annual cost, allocated by utilization %

Project-Specific equipment (tooling, fixtures, one-time purchases) spreads CapEx and maintenance directly over annual units (EAU) rather than by utilization percentage.

Equipment utilization accounts for all time the equipment is occupied: cycle time during production, order setups, and line setups for all FGs.

3.4 Material Costs

Auto-generated table: rows = non-customer-supplied BOM items, columns = volume breaks. The annual purchasing quantity for each cell is calculated automatically from the BOM quantities and EAUs. Each cell contains:

  • The calculated annual purchasing quantity (for reference)
  • An editable $/unit cost field
  • An optional Source text field (supplier name, quote number, date, etc.)
  • A "→" push button to copy the source across all breaks for that material

Archive and Proximity Matching

Every entered cost is stored in an archive keyed by part and annual purchasing quantity. When the active purchasing quantity changes (e.g., volume mix changes), the system searches the archive for a close match:

  • A stored cost is reused if: stored_qty ∈ [0.9 × current_qty, 10 × current_qty]
  • Cells using an archived cost at a different quantity are highlighted yellow with a note showing the archived quantity
  • Cells with no cost on record are highlighted pink
  • Costs matching within 2% of the current quantity are shown without any flag

This allows a single quote to contain costs sourced at different volumes without losing prior data when volume assumptions change.

3.7 Cost Summary

Read-only matrix: finished goods as row sections, volume breaks as columns. For each FG × break combination:

  • Full cost breakdown: Material / Direct Labor (Run, Line Setup, Order Setup, Equipment) / Indirect Labor (Annual Run, Line Setup, Order Setup) / Subcontract
  • Total / Unit — sum of all cost categories
  • Annual Cost — Total/Unit × EAU
  • Margin % — editable per FG per break; stored and remembered across sessions
  • Sell Price / Unit — calculated as Total ÷ (1 − Margin/100) (gross margin basis)
  • Annual Sell — Sell Price × EAU

Cells where material costs are incomplete (one or more BOM items have no cost on file) are flagged in red with a ⚠ indicator. Totals at the bottom of the table show total order quantity and total annual units per break.

4. Data Model

All application state is a single JSON document. This is what is saved to local storage and what is exported/imported via the JSON button. The structure:

{
  quote: { name, customer, date, revision, notes },
  // notes = HTML string, may contain base64-encoded images

  settings: { shopRate, indirectRate, capexYears, workingHoursPerYear },

  breaks: [{ id, label, buildsPerYear, totalEAU }],
  // totalEAU: optional target for the sum of all FG EAUs at this break

  finishedGoods: [{
    id, name, description,
    breaks: [{ eau }]   // index-aligned to breaks[]
  }],

  bom: [{
    id, partNumber, description, uom,
    fgSpecific: boolean,            // false = common, true = FG-specific
    customerSupplied: boolean,
    qty: number,                    // used when fgSpecific = false
    fgQtys: { [fgId]: number }      // used when fgSpecific = true
  }],

  materialCosts: {
    [bomItemId]: [{ annualQty, cost, timestamp }]   // cost archive per part
  },
  materialSources: {
    [bomItemId + "|" + breakIndex]: string          // source text per cell
  },

  equipment: [{
    id, name,
    capex: number,
    hourlyRunCost: number,
    annualMaintenance: number,
    projectSpecific: boolean
  }],

  directOps: [{
    id, name, operators, cycleTimeSec,
    orderSetupMin, lineSetupMin,
    equipmentIds: [equipmentId, ...],
    notes
  }],

  indirectOps: [{
    id, name, annualHours, orderSetupHrs, lineSetupHrs, notes
  }],

  subcontracts: [{
    id, name, priceEach, pricePerLine, pricePerOrder, pricePerYear, notes
  }],

  margins: { [fgId + "|" + breakIndex]: number }   // gross margin %
}

5. Calculation Rules

5.1 Volume Fundamentals

QtyPerBuild(FG_i, Break_j) = EAU[i][j] ÷ BuildsPerYear[j] TotalOrderQty(Break_j) = Σ_i QtyPerBuild(FG_i, Break_j) TotalAnnualUnits(Break_j) = Σ_i EAU[i][j] AnnualPurchQty(BOMItem_k, j) = Σ_i BOMQty(BOMItem_k, FG_i) × EAU[i][j] [customer-supplied items: always 0]

5.2 Material Cost Per Unit

MaterialCost(FG_i, Break_j) = Σ_k BOMQty(BOMItem_k, FG_i) × UnitCost(BOMItem_k, Break_j)

UnitCost is sourced from the material cost archive via proximity matching (see §3.4). If no match exists, the material subtotal is marked incomplete and flagged in the Summary.

5.3 Equipment Cost Per Unit

Calculated per operation, then summed. For each operation Op and each assigned equipment item Eq at break j:

OccupiedHrs/yr = (CycleTimeSec ÷ 3600 × TotalAnnualUnits[j]) + (OrderSetupMin ÷ 60 × BuildsPerYear[j]) + (LineSetupMin ÷ 60 × BuildsPerYear[j] × NumberOfFGs) Utilization = OccupiedHrs/yr ÷ WorkingHoursPerYear Non-Project-Specific equipment: CapExPerUnit = (Eq.capex ÷ CapExYears) × Utilization ÷ TotalAnnualUnits[j] MaintPerUnit = Eq.annualMaintenance × Utilization ÷ TotalAnnualUnits[j] Project-Specific equipment (tooling, fixtures, dedicated): CapExPerUnit = Eq.capex ÷ TotalAnnualUnits[j] MaintPerUnit = Eq.annualMaintenance ÷ TotalAnnualUnits[j] RunCostPerUnit = Eq.hourlyRunCost × (CycleTimeSec ÷ 3600) [run hours only, no setups] EquipCostPerUnit(Eq) = CapExPerUnit + MaintPerUnit + RunCostPerUnit

The utilization formula charges equipment for all time it is occupied: running parts, order setups, and line setups for every FG. This means a machine that does 10-minute order setups 12 times per year has that time counted against its annual capacity.

5.4 Direct Labor Cost Per Unit

RunCost(Op) = (CycleTimeSec ÷ 3600) × ShopRate × Operators LineSetupCost(Op) = (LineSetupMin ÷ 60) × ShopRate × Operators ÷ QtyPerBuild(FG_i, j) OrderSetupCost(Op) = (OrderSetupMin ÷ 60) × ShopRate × Operators ÷ TotalOrderQty(j) EquipCost(Op) = Σ_eq EquipCostPerUnit(eq) DirectLabor(FG_i, j) = Σ_op [ RunCost + LineSetupCost + OrderSetupCost + EquipCost ]

5.5 Indirect Labor Cost Per Unit

RunCost(Cat) = AnnualHours × IndirectRate ÷ TotalAnnualUnits[j] LineSetupCost(Cat) = LineSetupHrs × BuildsPerYear[j] × IndirectRate ÷ EAU[i][j] OrderSetupCost(Cat) = OrderSetupHrs × IndirectRate ÷ TotalOrderQty[j] Why TotalOrderQty for order setup: annual order setup cost = OrderSetupHrs × BuildsPerYear × Rate. Per-unit = ÷ TotalAnnualUnits = ÷ (TotalOrderQty × BuildsPerYear). The BuildsPerYear cancels, leaving OrderSetupHrs × Rate ÷ TotalOrderQty. This produces identical per-unit cost for any break with the same units-per-build ratio, regardless of how many times per year it runs. IndirectLabor(FG_i, j) = Σ_cat [ RunCost + LineSetupCost + OrderSetupCost ]

5.6 Subcontract Cost Per Unit

SubCost(FG_i, j) = Σ_s [ PriceEach + PricePerLine ÷ QtyPerBuild(FG_i, j) + PricePerOrder ÷ TotalOrderQty(j) + PricePerYear ÷ TotalAnnualUnits(j) ]

5.7 Total Cost and Sell Price

TotalCost(FG_i, j) = Material + DirectLabor + IndirectLabor + SubCost SellPrice(FG_i, j) = TotalCost ÷ (1 − Margin/100) [gross margin basis] AnnualSell(FG_i, j) = SellPrice × EAU[i][j]

Gross margin means the margin is expressed as a percentage of the sell price, not the cost. A 30% margin means 30 cents of every sell-price dollar is margin. This is the standard convention in manufacturing and distribution.

5.8 Takt Time

TaktSec = WorkingHoursPerYear × 3,600 ÷ max(TotalAnnualUnits[j] across all j)

Takt uses the highest-demand break as the binding constraint. If any direct labor operation has a cycle time exceeding takt, it is listed by name in the warning banner on the Operations tab.

6. UX Behavior

  • 7 tabs: Quote Info, Finished Goods, BOM, Material Costs, Equipment, Operations, Summary
  • Operations tab contains three sections: Direct Labor → Subcontracts → Indirect Labor
  • All sortable tables support drag-to-reorder with a blue line indicator showing the drop target; dragging to the first or last position is supported
  • Add buttons add exactly one row per click
  • Undo (up to 40 states) available in the header toolbar; no confirmation dialogs — errors shown inline, actions are reversible
  • State auto-saves to browser storage on every change; Export and Import JSON buttons in the header for file-based backup and sharing
  • Every user-facing input has an ⓘ icon with a descriptive tooltip (see §7)
  • Quote Notes is a rich content field; pasting an image (Ctrl+V) embeds it inline
  • Notes fields in Operations are textareas supporting multi-line input
  • BOM qty fields accept fractions at input (e.g. "1/10" → 0.1); the value is committed when the field loses focus, not on each keystroke
  • Equipment selector: searchable dropdown with checkboxes; selected items shown as chips; typing a new name not in the list and selecting "Add as new" simultaneously creates the equipment entry and assigns it; chips for items with no defined cost attributes shown in orange with ⚠
  • Takt notice and all cost calculations update live as values are changed anywhere in the app
  • Material cost source field per cell; push button copies source to all breaks for that material
  • Summary margin inputs are per-FG per-break; entered values persist across sessions
