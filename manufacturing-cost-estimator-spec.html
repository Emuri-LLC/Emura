1. Overview

The Manufacturing Cost Estimator is a standalone, offline, single-file HTML application for estimators at contract manufacturing companies. It runs entirely in a browser with no server, no installation, and no internet required.

The application aggregates four cost categories across multiple finished goods and volume scenarios:

  • BOM / Material Costs — raw materials and purchased components
  • Direct Labor — work-cell operations with cycle times, setup times, and equipment costs
  • Indirect Labor — overhead categories (engineering, quality, purchasing, etc.)
  • Subcontract Costs — outsourced operations or services

1.1 Design Goals

  • Simple, fast, flexible — no frameworks, no build step, no dependencies
  • Handles multi-FG quotes with different volume mixes per finished good
  • Clean separation of order-level vs. line-level setup costs throughout
  • Volume-intelligent material cost lookup with archive and proximity matching
  • All data persists in browser localStorage; import/export via JSON file
  • Every input has a tooltip (ⓘ) explaining purpose and expected value

1.2 Non-Goals

  • ERP/MRP integration, CRM, multi-user access, inventory, accounting

2. Key Concepts

2.1 Line vs. Order

LineAn individual production job for a single FG. Each FG occupies its own line. Line setup time is specific to the FG and incurred once per order event per FG.
OrderA build event where all lines run simultaneously — one line per FG. Order setup time is incurred once per event and its cost is allocated across all units in the order.

2.2 Volume Breaks

Volume break columns represent demand scenarios. Each column has a Builds Per Year (applies to all FGs) and an optional Target Total EAU. Each FG independently has an EAU per column. The FG EAU sum is validated against the target and can be auto-populated using mix proportions.

2.3 Work Cell

A work cell is a direct labor operation with a fixed operator count and a single cycle time (seconds/unit). The cell produces one unit per cycle. Cycle time governs throughput and must be ≤ takt time to meet demand on a single shift.

2.4 Takt Time

Takt = (Working Hours/Year × 3600) ÷ (Total Annual Units at highest-demand break). Displayed in the Operations tab. Operations with cycle times exceeding takt are flagged.

2.5 Equipment

Equipment is defined on the Equipment tab and assigned to one or more direct labor operations. Each piece of equipment contributes three cost types:

  • CapEx: time-based allocation via utilization %
  • Hourly Run Cost: variable cost per run hour (cycle time only)
  • Annual Maintenance: like CapEx but amortized over 1 year

Project-Specific equipment spreads CapEx and maintenance directly over EAU instead of by utilization %.

3.4 Material Costs

Auto-generated table: rows = non-customer-supplied BOM items, columns = volume breaks. Each cell shows the calculated annual purchasing quantity and contains:

  • An editable $/unit cost field
  • An optional Source text field (supplier name, quote #, date)
  • A "→" push button to copy the source across all breaks for that material

Archive & Proximity Matching

  • All entered costs are stored in an archive keyed by (RM ID, annual purchasing quantity)
  • An archived cost is reused if: archived_qty ∈ [0.9× current_qty, 10× current_qty]
  • Flagged cells (archive at different qty) shown in yellow; missing costs shown in pink

5. Calculation Rules

5.1 Volume Fundamentals

QtyPerBuild(FG_i, Break_j) = EAU_i_j ÷ BuildsPerYear_j TotalOrderQty(Break_j) = Σ QtyPerBuild(FG_i, Break_j) TotalAnnualUnits(Break_j) = Σ EAU_i_j AnnualPurchQty(RM_k, Break_j) = Σ BOMQty(RM_k, FG_i) × EAU_i_j [customer-supplied: 0]

5.2 Material Cost Per Unit

MaterialCost(FG_i, Break_j) = Σ BOMQty(RM_k, FG_i) × UnitCost(RM_k, Break_j)

5.3 Equipment Cost Per Unit

For each operation Op and each assigned equipment item Eq:

OccupiedHrs_per_yr = (CycleTimeSec/3600 × TotalAnnualUnits) + (OrderSetupMin/60 × BuildsPerYear) + (LineSetupMin/60 × BuildsPerYear × NumFGs) Utilization = OccupiedHrs_per_yr ÷ WorkingHoursPerYear If NOT Project-Specific: CapExPerUnit = (Eq.capex ÷ CapExYears) × Utilization ÷ TotalAnnualUnits MaintPerUnit = Eq.annualMaintenance × Utilization ÷ TotalAnnualUnits If Project-Specific: CapExPerUnit = Eq.capex ÷ TotalAnnualUnits MaintPerUnit = Eq.annualMaintenance ÷ TotalAnnualUnits RunCostPerUnit = Eq.hourlyRunCost × (CycleTimeSec ÷ 3600) [cycle time only] EquipCostPerUnit = CapExPerUnit + MaintPerUnit + RunCostPerUnit

5.4 Direct Labor Cost Per Unit

RunCost = (CycleTimeSec ÷ 3600) × ShopRate × Operators LineSetupCost = (LineSetupMin ÷ 60) × ShopRate × Operators ÷ QtyPerBuild OrderSetupCost= (OrderSetupMin ÷ 60) × ShopRate × Operators ÷ TotalOrderQty EquipCost = Σ EquipCostPerUnit (all assigned equipment) DirectLabor = RunCost + LineSetupCost + OrderSetupCost + EquipCost

5.5 Indirect Labor Cost Per Unit

RunCost = AnnualHours × IndirectRate ÷ TotalAnnualUnits LineSetupCost = LineSetupHrs × BuildsPerYear × IndirectRate ÷ EAU_i OrderSetupCost = OrderSetupHrs × IndirectRate ÷ TotalOrderQty Note: Equivalent to (OrderSetupHrs × BuildsPerYear × IndirectRate) ÷ TotalAnnualUnits — bpy cancels because TotalAnnualUnits = TotalOrderQty × BuildsPerYear

5.6 Subcontract Cost Per Unit

SubCost = PriceEach + PricePerLine ÷ QtyPerBuild + PricePerOrder ÷ TotalOrderQty + PricePerYear ÷ TotalAnnualUnits

5.7 Total and Sell Price

TotalCost = Material + DirectLabor + IndirectLabor + SubCost SellPrice = TotalCost ÷ (1 − Margin/100) [gross margin basis]

5.8 Takt Time

TaktSec = WorkingHoursPerYear × 3600 ÷ max(TotalAnnualUnits across all breaks)

6. UX Requirements

  • 7 tabs: Quote Info, Finished Goods, BOM, Material Costs, Equipment, Operations, Summary
  • Operations tab contains three sections: Direct Labor → Subcontracts → Indirect Labor
  • All sortable tables support HTML5 drag-to-reorder; drop indicator is a colored line between rows; dragging to the end of a list is supported
  • Add buttons add exactly one row per click; structural changes push an undo state
  • Undo (up to 40 states) available in header; no confirmation dialogs — errors shown inline
  • Auto-save to localStorage on every change (300ms debounce); export/import JSON
  • Every user input has an ⓘ icon with a descriptive tooltip (see §7)
  • Quote Notes is a contenteditable div; pasting an image embeds it as base64
  • Operations notes fields are textareas supporting multi-line input
  • BOM qty accepts fractions (e.g. "1/10" → 0.1, rounded to 4 decimal places)
  • Equipment selector: searchable dropdown with checkboxes; selected items shown as chips; typing a new name and selecting it creates a new equipment entry; chips for items with no defined attributes shown in orange with ⚠
  • Takt notice updates immediately when cycle time is changed (without tab switch)
  • Material costs source field per cell; push button copies source to all breaks for that material
